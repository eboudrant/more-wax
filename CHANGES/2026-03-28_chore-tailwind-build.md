# chore: replace Tailwind CDN with compiled CSS

**Date:** 2026-03-28
**Type:** Chore

## Intent

Remove the Tailwind CDN runtime script (~300KB JS) and replace it with a pre-compiled static CSS file (~42KB). Eliminates the browser console warning about CDN usage in production and improves page load performance.

### Prompts summary

1. Replace Tailwind CDN with build-time compilation
2. Add CI check to catch uncommitted CSS changes

## Changes

### `tailwind.config.js` (new)
- Extracted inline config from `index.html` to a proper config file
- Content scanning: `static/index.html` and `static/js/**/*.js`
- Plugins: `@tailwindcss/forms`, `@tailwindcss/container-queries`
- Custom colors, fonts from the existing design system

### `static/src/input.css` (new)
- Tailwind directives (`@tailwind base/components/utilities`)
- All custom component CSS from the old `static/styles.css` (modals, buttons, scanner, toast, wall view, etc.)

### `static/styles.css`
- Now a build artifact generated by `npm run css:build`
- Contains compiled Tailwind utilities + custom component CSS, minified

### `static/index.html`
- Removed CDN `<script>` tag and inline `tailwind.config` block
- Replaced with `<link rel="stylesheet" href="/static/styles.css">`
- Removed duplicate stylesheet reference

### `package.json`
- Added `tailwindcss`, `@tailwindcss/forms`, `@tailwindcss/container-queries` as devDependencies
- Added `css:build` and `css:watch` scripts

### `.github/workflows/ci.yml`
- Added `CSS Build` job: installs deps, rebuilds CSS, fails if `styles.css` differs from committed version

### Bandit fixes
- Added `# nosec B310` to new `urlopen` in `handler.py` (Google OAuth validation)
- Cleaned up verbose `nosec` comments that bandit misinterpreted as test IDs

## Dev workflow

```bash
# One-time build
npm run css:build

# Watch mode during development
npm run css:watch

# After changing Tailwind classes in HTML/JS, rebuild and commit styles.css
```
