# feat: internationalization (i18n) ŌĆö EN, JA, FR, DE

**Date:** 2026-04-08
**Type:** Feature

## Intent

Translate the entire app into Japanese, French, and German. Lightweight custom i18n system with no external dependencies ŌĆö just a `t()` function, JSON translation files, and `data-i18n` attributes.

### Prompts summary

1. Scalable i18n without build step or npm dependencies
2. `t()` function with ICU-lite plurals and interpolation
3. Language detection from browser + manual override in Settings
4. Incremental migration ŌĆö app works at every step

## Architecture

- `static/js/i18n.js` ŌĆö global `t(key, vars)` function, `setLocale()`, `translateDOM()`
- `static/locales/{en,ja,fr,de}.json` ŌĆö 287 translation keys each
- `data-i18n` attributes on HTML elements for static text
- `t()` calls in JS for dynamic text
- ICU-lite plural rules: `{count, plural, one {# record} other {# records}}`
- CLDR plural rules hardcoded per language (one-liner each)
- English always loaded as fallback ŌĆö missing keys fall through gracefully
- Language switcher in Settings modal with flag emojis
